## Create Key

<api>POST /v1/admin/keys</api>

<ParamField body="name" type="string" required>
  Descriptive name for the key.
</ParamField>

<ParamField body="role" type="string" required>
  One of `admin`, `readwrite`, or `readonly`.
</ParamField>

<ParamField body="expires_in_days" type="integer">
  Number of days until the key expires. Omit for no expiry.
</ParamField>

<RequestExample>
```bash cURL
curl -X POST http://localhost:8000/v1/admin/keys \
  -H "x-api-key: your-admin-key" \
  -H "Content-Type: application/json" \
  -d '{"name": "production-app", "role": "readwrite", "expires_in_days": 90}'
```
</RequestExample>

<ResponseExample>
```json Success
{
  "status": "success",
  "data": {
    "id": 2,
    "name": "production-app",
    "key": "sk-rw-abc123xyz...",
    "role": "readwrite",
    "is_active": true,
    "created_at": "2024-01-15T10:00:00Z",
    "expires_at": "2024-04-15T10:00:00Z",
    "last_used_at": null
  },
  "error": null
}
```
</ResponseExample>

<Warning>
  The `key` value is only returned at creation and rotation. Copy it immediately — it cannot be retrieved later.
</Warning>

## Update Key

<api>PATCH /v1/admin/keys/{key_id}</api>

Update a key's name, role, or active status. Use `is_active: false` to revoke, `is_active: true` to restore.

<ParamField path="key_id" type="integer" required>
  The numeric ID of the key.
</ParamField>

<ParamField body="name" type="string">
  New name for the key.
</ParamField>

<ParamField body="role" type="string">
  New role. One of `admin`, `readwrite`, `readonly`.
</ParamField>

<ParamField body="is_active" type="boolean">
  Set to `false` to revoke the key, `true` to restore it.
</ParamField>

<RequestExample>
```bash Revoke
curl -X PATCH http://localhost:8000/v1/admin/keys/2 \
  -H "x-api-key: your-admin-key" \
  -H "Content-Type: application/json" \
  -d '{"is_active": false}'
```
</RequestExample>

## Rotate Key

<api>POST /v1/admin/keys/{key_id}/rotate</api>

Regenerate the key value. The old key immediately stops working.

<ParamField path="key_id" type="integer" required>
  The numeric ID of the key to rotate.
</ParamField>

<ResponseExample>
```json Success
{
  "status": "success",
  "data": {
    "id": 2,
    "name": "production-app",
    "key": "sk-rw-new-secret-value...",
    "role": "readwrite",
    "is_active": true,
    "rotated": true,
    "created_at": "2024-01-15T10:00:00Z",
    "expires_at": "2024-04-15T10:00:00Z",
    "last_used_at": null
  },
  "error": null
}
```
</ResponseExample>
