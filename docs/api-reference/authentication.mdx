## User Registration & Login

The auth endpoints do **not** require an `x-api-key` header.

### Register

```bash
curl -X POST http://localhost:8000/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "password": "securepass"}'
```

Response:

```json
{
  "status": "success",
  "data": {
    "user": { "id": 1, "email": "user@example.com", "created_at": "..." },
    "api_key": { "id": 1, "key": "sk-...", "name": "user@example.com-admin", "role": "admin" }
  }
}
```

### Login

```bash
curl -X POST http://localhost:8000/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "password": "securepass"}'
```

Returns the same shape as register. Use the returned `api_key.key` for all subsequent requests.

### Multi-Tenancy

Each user's API keys and collections are isolated. A key can only see and modify resources belonging to the same user. The bootstrap admin key (from `API_KEY` env var) has global access.

## Creating API Keys

API key management requires an `admin`-role key.

```bash
# Create a readwrite key
curl -X POST http://localhost:8000/v1/admin/keys \
  -H "x-api-key: your-admin-key" \
  -H "Content-Type: application/json" \
  -d '{"name": "production-app", "role": "readwrite"}'
```

Response:

```json
{
  "status": "success",
  "data": {
    "id": 2,
    "name": "production-app",
    "key": "sk-rw-abc123...",
    "role": "readwrite",
    "created_at": "2024-01-15T10:00:00Z"
  }
}
```

<Warning>
  The API key value is only shown once at creation time. Store it securely.
</Warning>

## Listing API Keys

```bash
curl http://localhost:8000/v1/admin/keys \
  -H "x-api-key: your-admin-key"
```

## Revoking an API Key

```bash
curl -X DELETE http://localhost:8000/v1/admin/keys/2 \
  -H "x-api-key: your-admin-key"
```

## Key Roles

<ParamField body="role" type="string">
  One of `admin`, `readwrite`, or `readonly`.

  - **admin** — full access, including key management
  - **readwrite** — create/delete collections, upsert/delete vectors, search
  - **readonly** — search, list, and get operations only
</ParamField>

## Using API Keys

Pass the key in the `x-api-key` header on every request:

```bash
curl http://localhost:8000/v1/collections \
  -H "x-api-key: sk-rw-abc123..."
```

Or set it once in your SDK client:

```python
from vectordb_client import VectorDBClient

client = VectorDBClient(
    base_url="http://localhost:8000",
    api_key="sk-rw-abc123...",
)
```
