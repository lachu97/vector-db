---
title: Hybrid Search
api: POST /v1/collections/{collection_name}/hybrid_search
description: Combine vector similarity and keyword matching using Reciprocal Rank Fusion.
---

<ParamField path="collection_name" type="string" required>
  Collection to search.
</ParamField>

<ParamField header="x-api-key" type="string" required>
  API key with any role.
</ParamField>

<ParamField body="query_text" type="string" required>
  Text query for keyword matching.
</ParamField>

<ParamField body="vector" type="number[]">
  Query vector for semantic similarity. Now optional -- if omitted, the backend auto-embeds `query_text` to generate the vector.
</ParamField>

<ParamField body="k" type="integer" default="10">
  Number of results to return.
</ParamField>

<ParamField body="offset" type="integer" default="0">
  Pagination offset.
</ParamField>

<ParamField body="alpha" type="float" default="0.5">
  Blend factor. `1.0` = pure vector search, `0.0` = pure keyword search, `0.5` = equal weight.
</ParamField>

<ParamField body="filters" type="object">
  Optional metadata filter.
</ParamField>

<ParamField body="include_timing" type="boolean" default="false">
  When `true`, the response includes a `timing_ms` object with `embedding_ms`, `search_ms`, and `total_ms` breakdowns.
</ParamField>

<RequestExample>
```bash cURL (with vector)
curl -X POST http://localhost:8000/v1/collections/articles/hybrid_search \
  -H "x-api-key: your-key" \
  -H "Content-Type: application/json" \
  -d '{
    "query_text": "machine learning transformers",
    "vector": [0.1, 0.2, 0.3],
    "k": 10,
    "alpha": 0.7
  }'
```

```bash cURL (text only, auto-embed)
curl -X POST http://localhost:8000/v1/collections/articles/hybrid_search \
  -H "x-api-key: your-key" \
  -H "Content-Type: application/json" \
  -d '{
    "query_text": "machine learning transformers",
    "k": 10,
    "alpha": 0.7,
    "include_timing": true
  }'
```
</RequestExample>
